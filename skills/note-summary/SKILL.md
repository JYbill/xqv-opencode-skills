# Note Summary

将截图、板书、课件、流程图、结构图这类内容整理成适合复习和二次复制的文本笔记。

## Purpose

从图片里提取文字信息。

重建图片旁边的结构关系。

把零散视觉信息整理成适合记笔记的纯文本输出。

默认输出 ASCII 框图加文字解释，保证可以直接复制到文档、备忘录、Markdown 或聊天记录里。除非用户明确要求纯文本，否则不要省略框图。

## When this skill is useful

优先处理下面这几类请求。

把图片转成文字。

把图和文字一起整理成笔记。

把课件里的结构图改写成纯文本。

把寻址方式、流程图、层级图、关系图整理成可复制框图。

按 标题 → 框图 → 解释 的顺序输出。

把内容改成考试复习稿、背诵稿、课堂笔记稿。

即使用户只说整理成笔记，只要图片里存在明显结构关系，也默认返回 框图 + 文字解释。

## Core workflow

先完整读取图片。

先辨认能稳定识别的文字，再补结构，不要一上来猜细节。

如果图片同时有左侧说明和右侧结构图，分别提取，再合并成一份统一笔记。

优先保留原图里的术语。

遇到模糊区域时，只输出高置信内容。对看不清的字，直接说明模糊，不要脑补专业名词。

如果用户要的是笔记版，默认给 标题 + ASCII 框图 + 解释；只有在用户明确说只要纯文本时，才省略框图。

如果用户要的是框图版，优先给 ASCII 框图。

如果用户对格式做过纠正，后续输出严格沿用那个顺序。

## Output modes

根据用户措辞选择最贴近的输出。

### 纯文本笔记

适合 将图片转文本、整理成文字、做成笔记 这类请求。

输出目标是把图中文字和图形含义合成可读文本。

建议结构如下。

```text
标题

核心内容整理
含义
特点
补充说明
```

### 框图笔记

适合 返回一个框图、画成文字框图、用文本重画结构图 这类请求。

默认使用下面顺序。

先写标题。

换行后给单个框图。

再换行写解释。

不要先讲解释再给图。

不要把多个知识点堆成一大页，除非用户明确要求总览版。
默认一个知识点对应一个单独框图，不要把多个寻址方式合并成总表。

推荐模板如下。

```text
寄存器间接寻址

┌────┬────┐
│ OP │ R  │
└────┴────┘
       │
       ▼
   ┌────────┐
   │ 寄存器 │ ─────→ 操作数地址
   └────────┘               │
                            ▼
                        ┌────────┐
                        │ 主存   │ ───→ 操作数
                        └────────┘

含义是……。特点是……。
```

### 复习稿

适合 帮我整理成背诵版、考试笔记版、超简版 这类请求。

保留最少必要信息。

把一句一义写清楚。

避免堆细节。

## Diagram reconstruction rules

将视觉关系翻译成文本关系时，优先保留这几种元素。

方框。

箭头方向。

层级关系。

先后访问顺序。

中间量和最终量的区别。

例如这类链路要明确分层。

指令 → 地址码 → 主存 → 操作数

指令 → 地址码 → 主存 → 操作数地址 → 主存 → 操作数

指令 → 寄存器编号 → 寄存器 → 操作数

指令 → 寄存器编号 → 寄存器 → 操作数地址 → 主存 → 操作数

框图不需要追求和原图像素级一致。

重点是关系正确、顺序正确、术语正确、复制方便。

## OCR judgment rules

对清晰内容直接整理。

对模糊标题，只有在上下文足够强时才谨慎补全。

对无法确认的字，不要装作看清了。

如果只是个别词模糊，但整体结构很清楚，可以在说明里点一下 某词较模糊，结构关系已按清晰部分整理。

如果用户后续纠正了某个术语或格式，以用户版本为准，立即沿用。

## Style rules

默认简洁。

不要输出 OCR 过程。

不要重复同一个信息。

不要为了显得完整而加入图片里没有的知识扩展。

用户没要求时，不额外讲背景课本知识。

用户要的是可复制笔记，不是图像分析报告。

## Response patterns

### Pattern A: 用户只要文本

直接返回整理后的纯文本笔记，不画框图。

### Pattern B: 用户说整理成笔记，但没限制格式

默认返回 标题 + 单个或多个必要框图 + 解释。

如果一张图里包含多个知识点，每个知识点都要拆成单独小节，分别给标题、框图、解释，不要先画总览树再展开。

如果图片同时有左侧说明和右侧结构图，合并成统一笔记。

### Pattern C: 用户要 图和文字都转成文本

默认理解为要可复制笔记，返回 框图 + 文字解释。

先给术语标题。

再给对应框图或结构链路。

再给含义或特点。

### Pattern D: 用户要 单个框图

只输出一个标题对应一个框图。

框图后紧跟说明段。

不要顺手把其他同类知识点一起展开，除非用户明确要求全集。

### Pattern E: 用户给出格式纠正

把纠正当成最终模板。

后续严格按该模板输出。

例如用户要求 先标题，再框图，再介绍 时，后续就固定使用这个顺序。

## Final check

检查标题是否先于框图出现。

检查框图是否只有当前用户要的那一个知识点。

检查解释是否放在框图之后。

检查术语是否与图片一致。

检查输出是否能直接复制进笔记软件。
